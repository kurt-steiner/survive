# 内存管理

## 分页管理

### 地址变换

我们有一台 32位的机器，地址以字节为单位，且在分页管理中，**页面大小与 内存块/行大小相同**，为 4KB  
在这种情况下，能表示 $2^{32}$ 个内存地址，可以表示 $2^{32} / 4KB = 2^{20}$ 个页面，因此我们用 20 位二进制数表示页号  
而一页的大小为 $2^{12}B$ ，因此，我们用 12 位二进制数表示页内偏移

```rust
const PAGE_SIZE: u32 = 4096; //页面大小，4096kb
```

在将 **逻辑地址** 转换为 **物理地址的时候** ，需要将 逻辑地址 转换成 页号 和 页内偏移，我们用这个结构来表示

```rust
struct Address {
    offset_page: usize, // 页号
    offset_in_page: u32 // 页内偏移
}

impl Address {
    fn from(logic_address: u32) -> Self {
        let offset_page = logic_address / PAGE_SIZE;
        let offset_in_page = logic_address % PAGE_SIZE;

        Self {
            offset_page: offset_page as usize,
            offset_in_page
        }
    }
}
```

我们将 逻辑地址 转换为 `Address` 结构，其中用到

1. 整除 页面大小 获取 页号(0-based)
2. 取模 页面大小 获取 页内偏移(0-based)

那又如何使用这个结构获取 物理地址呢？  
我们知道，一个进程中有他自己的页表，这个页表可以看作一个数组，其中的元素存储了物理块号

```rust
struct PageTable {
    length: u32, // 页表长度
    data: [u32; PAGE_TABLE_SIZE] // 页表数据，对应 map<页号,内存块号>
}

struct Progress {
    page_table: PageTable
}
```

在这个进程中使用逻辑地址 寻址时

```rust
impl Progress {
    fn find(&self, logic_address: u32) -> Option<u32> {
        let address = Address::from(logic_address);
        if address.offset_page >= self.page_table.length as usize {
            return None;
        }

        let offset_page = address.offset_page;
        let offset_memory_block = self.page_table.data[offset_page]; // 内存块号
        let address_memory_block = offset_memory_block * PAGE_SIZE; // 内存块号地址
        let raw_address = address_memory_block + address.offset_in_page;
        return Some(raw_address);
    }
}
```

我们需要

1. 转换逻辑地址 为 `Address` ，其中带有 页号 和 页内偏移
2. 判断 页号 是否超出了 索引范围，如果超出，发出越界中断
3. 从 进程的页表 中取出对应 **页号/索引** 的数据，这个数据存储的是内存块号
4. 内存块号 x 内存块大小 获取 内存块号的地址，由于本例中 内存块大小和页面大小 相同，内存块号地址 可以看作 内存块号 x 页面大小
5. 对内存块号地址 加上 页内偏移，得到真实物理地址

上述我们是用软件的视角去看待 地址变换 的过程，在硬件实现中略有不同，尤其是对 页表的访问  
在软件视角下，我们已经知道了 **页表的长度** 用作判断 越界访问，但在硬件视角下，我们还需要知道 页表对象 所在的 **内存首地址**

我们再次整理整个过程，对一个逻辑地址 `logic_address` 进行寻址，硬件会

1. 计算页号 `offset_page = logic_address / PAGE_SIZE`
2. 计算页内偏移 `offset_in_page = logic_address % PAGE_SIZE`
3. 从进程页表中获取存储的**内存块号**
    - 判断是否发生缺页中断  
    `offset_page >= length(page_table)`
    - 计算页表项地址  
    `address_page_entry = address(page_table) + offset_page * PAGE_SIZE`
    - 获取页表项内容，即内存块号  
    `offset_memory_block = *address_page_entry`
4. 获取内存块号 对应的物理地址  
    `address_memory_block = offset_memory_block * MEMORY_BLOCK_SIZE`  
    或者  
    `address_memory_block = offset_memory_block * PAGE_SZE`
5. 对内存块号地址 加上 页内偏移，获取真实地址  
    `address = address_memory_block + offset_in_page`
