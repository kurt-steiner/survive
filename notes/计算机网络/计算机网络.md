# 计算机网络

## 对层次进行划分

在计算机网络这本书中，我们发现目录有 物理层，数据链路层，网络层，传输层，应用层 的内容，现在我们已经大致学完了所有内容，开始对整个计算机网络进行整理  
我们画这样一个图来表示这些章节  
![image](../images/计算机网络.png)

如果我们从工业原料运输的视角来看

1. 首先我们需要从原料场地 采集原料
2. 现在我们要把原料运输到客户仓库中
3. 在运输前，我们要确定 **从出发地到客户** 的最佳路径
4. 在运输前，由于货物太多，我们需要组建车队来运输
5. 在运输的路径中，有许多站点可供车辆停靠，可以用来补给，司机太累也可以进行任务交接
6. 在运输途中，由于车队庞大，难免造成 整条路径 拥堵，要时刻避免这种情况
7. 运输到客户后，客户可以随意拿原料进行加工

但是在计算机网络中，物理层可不是数据产生的地方，用户的程序才是，我们调整下这个类比

我们要在两个设备之间传输数据，但是两方之间的硬件设备可能不一样，两方之间对电信号的处理逻辑也不一致，于是我们抽象出一层，叫作**数据链路层**，他需要屏蔽底层物理特性，使得数据能够在两个节点之间传输  
当数据成功传输到接收方，接收方在应用层接可以对其进行处理

可以看到，物理层和应用层 是两个极端，一个服务于设备的硬件，一个服务与设备的软件  
在网络中负责传输数据的是 数据链路层，网络层和传输层 这三个东西，我们要对其进行重点探讨

## 从数据运输的全局来看

现在我们要进行货物运输，把浙江温州江南皮革厂的货物 运输到 浙江杭州电子科技大学  
我们要知道，现在没有这两个地方的直通车，我们通常是

1. 查下导航，判断最佳路径为 **温州南站 - 杭州东站 - 杭州地铁一号线/萧山国际机场方向 - 文泽路站B口 - 步行 - 杭州电子科技大学**
2. 我们先打车去 温州南站
3. 经过动车运输，到达杭州东站
4. 坐杭州地铁一号线/萧山国际机场方向
5. 从 文泽路站B口 出地铁
6. 步行到 杭州电子科技大学

可以看到，从全局来看，出发地 到 目的地 只有一条路径，两个节点  
从局部来看，这条路径中由许多节点构成  
这就好像一条铁链，两端之间有许多部件，将整个铁链组合在一起，

我们知道 **传输层** 和 **数据链路层** 之间都有 可靠传输，差错控制 和 流量控制，他们之间的功能其实不冲突  

1. 传输层 关注的是 发送端/接收端 两方的传输，是全局视角
2. 数据链路层 关注的是整条路径中各个节点之间的传输，是局部视角  

这也是为什么拿铁链比作数据链路层的原因，其就像一条链 将各个站点结合在一起，组成一条路径

___
但是，理论和现实总是有差别的，我们在整个运输过程中，是 **集中的，预先规划的**  
但在网络中，是 **分布式的，逐跳决策** 的  
也就是说，如果我们按照网络中的做法，应该是这样

我们只知道要从浙江温州江南皮革厂出发，去往杭州电子科技大学，但是我们没有导航，也不知道两地之间如何走，我们只能**问路**

1. 第一站: 温州南站/本局域网中路由器
    - 我们来到温州南站，问工作人员: "我要到杭州电子科技大学，请问该怎么走"
    - 工作人员查了下手里的 列车时刻表/路网图/路由表，根据目的地 **杭州** 判断，应该是先去 杭州东站
    - 于是我们买票去 杭州东站
2. 第二站: 杭州东站/网络中的路由器节点
    - 我们下车后问 杭州东站 的工作人员: "我要到杭州电子科技大学，请问该怎么走"
    - 工作人员查了下手里的 列车时刻表/路网图/路由表，告诉我们，应该乘坐 杭州地铁一号线/萧山国际机场方向，从文泽路站下车
    - 于是我们刷卡进站
3. 第三站: 文泽路站/网络中的路由器节点
    - 从文泽路站下车，我们向 地铁工作人员问路
    - 同样的，工作人员查看手里的 路网图/路由表，告诉我们，从B口出地铁，沿着路直行，就能走到目的地了
4. 第四站: 杭州电子科技大学/目的地
